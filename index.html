什么是 Clash？

透彻理解底层技术逻辑，规避概念误区，建立清晰的代理认知。

Clash 是一款以 Go 语言编写的开源的通用网络代理客户端和流量转发工具，主要用于管理代理连接、配置网络规则以及实现智能流量分流。它通过规则（Rule-Based）系统，让用户可以根据域名、IP 地址、应用请求等条件，自定义不同网络流量的处理方式。

客户端能为您做什么？核心功能解析

不只是代理，更是一种精细化的网络流量控制中枢。

1. 一键管理代理连接

Clash 客户端提供可视化操作界面，让用户轻松完成：导入代理订阅链接、添加和管理多个配置文件、快速切换代理线路、查看当前连接状态、启用或关闭系统代理

2. 智能规则分流

客户端可以根据不同网络请求自动选择连接方式，依据本地配置文件中预设的规则匹配（如域名匹配、IP段、地理位置、端口等），用户无需频繁手动切换模式，实现自动化网络管理。

3. 多代理节点统一管理

Clash 客户端支持集中管理多个代理节点（不同地区服务器、不同协议节点、多个订阅来源、自定义代理配置），并可以在客户端内快速切换。

4. 代理策略组管理

Clash 支持策略组功能，让用户创建更加灵活的网络策略（例如：根据延迟自动选择最佳线路、当当前节点不可用时自动切换）。

5. TUN 模式接管系统流量

最新版的Clash 客户端支持 TUN 模式，通过创建虚拟网络接口接管系统流量。

网络代理专有名词核心释义

拒绝信息差，用大白话和技术逻辑拆解网络调优中的核心概念。

什么是代理？

顾名思义，代理（Proxy） 是一种位于用户设备与目标互联网服务器之间的中间服务器。用户的网络请求不会直接发送给目标网站，而是先经过代理服务器，由代理服务器代替用户向目标服务器发起请求，再将返回的数据传递给用户。

什么是代理协议？

代理协议（Proxy Protocol） 是指客户端与代理服务器之间进行通信时所遵循的一套规则和标准。它规定了数据如何传输、如何建立连接、如何进行身份验证以及如何保护通信内容。常见的代理协议有 Shadowsocks、Vmess、Trojan 等。

什么是节点与机场？

在 Clash、V2Ray、Sing-box 等代理客户端的使用过程中，经常会看到“节点”“机场”这些词。节点是搭载了代理协议服务端程序的实际出口服务器（如香港1、新加坡1、美国1等），节点直接与本地的 Clash 内核建立通信。而当节点越来越多时，就需要一个集中配置来管理这些节点，这个配置就是订阅地址（由机场提供）。机场就是订阅的供应商，拥有官方网站供用户注册登录并获取对应的订阅凭证。

什么是 GFW？

GFW 是 Great Firewall（Great Firewall of China，中国国家防火墙） 的简称，是网络社区中对中国大陆互联网过滤和监管系统的俗称。

什么是 TUN 模式？

系统代理只是系统默认自带的一个入口，通常只能影响支持系统代理的应用。而 TUN 模式（TUN Mode） 是一种通过创建虚拟网络接口（Virtual Network Interface）来接管系统网络流量的代理模式，TUN 模式可以让代理客户端像一个“虚拟网卡”一样工作，接管设备的大部分网络请求，而不需要每个应用单独设置代理。

系统代理与 TUN 模式流量链路对比

系统代理链路：应用流量 → 系统代理 → 分流规则 → 出口节点
TUN 模式链路：应用流量 → TUN 虚拟网卡 → 分流规则 → 出口节点
同时开启链路：应用流量 → 系统代理 → TUN 虚拟网卡 → 分流规则 → 出口节点

什么是 DNS 污染及缓解思路？

DNS 污染是指用户发起域名解析请求时，返回的 DNS 结果被中途修改或干扰，导致用户获得错误的 IP 地址，无法正常访问目标服务。遇到 DNS 污染的缓解思路是：针对关键域名使用直连/代理优先的分流规则，避免被本地网络劫持；同时避免使用不可信的公共 Wi-Fi，或在不可信网络下先建立可靠的代理链路后再进行全量解析。
